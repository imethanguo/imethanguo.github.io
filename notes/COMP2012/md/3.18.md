## ABC (Abstract Base Class)

```cpp
class Shape {
public:
  virtual double get_square() const = 0; // pure virtual function
};
```

```cpp
class Square : public Shape {
public:
  Square() {}
  // If get_square() is not implemented, Square is still abstract.
};
```

<span style="color: red;">反例: Shape obj; （cannot create object of abstract class）</span><br>
<span style="color: red;">反例: Square obj; （error too if Square does not override get_square()）</span>

An ABC has two properties:

1. No objects of ABC can be created.
2. Derived classes must implement all pure virtual functions, otherwise they are also ABCs.

### ABC as an Interface

An abstract base class provides a uniform interface for different derived classes.

An abstract base class cannot be used:

1. As an argument type passed by value.
2. As a function return type returned by value.
3. As the type of an explicit conversion.

## `final` Keyword

```cpp
class A {};
class B : public A {};
class C final : public B {}; // cannot inherit from C anymore
class D : public B {};
```

<span style="color: red;">反例: class E : public C {}; （base class C is marked final）</span>

For virtual functions:

```cpp
class UPerson {
public:
  virtual void print() const {}
};

class Student : public UPerson {
public:
  // Derived classes of Student cannot override print again.
  void print() const override final {
    // ...
  }
};

class PG_Student : public Student {
public:
  void study() const {}
};
```

<span style="color: red;">反例: void print() const override {} （print is final in Student）</span>

## Function Template

Problem: many overloaded functions with same logic.

```cpp
int larger(int a, int b) { return (a > b) ? a : b; }
double larger(double a, double b) { return (a > b) ? a : b; }
// ... too many similar functions
```

Template solution:

```cpp
template <typename T>
const T& larger(const T& a, const T& b) {
  return (a > b) ? a : b;
}
```
