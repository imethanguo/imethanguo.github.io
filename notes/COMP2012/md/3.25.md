## Template Issue: Return by Reference

```cpp
template <typename T1, typename T2>
inline const T1& larger(const T1& a, const T2& b) {
  return (a < b) ? b : a;
}
```

Potential issue:

- If `b` is returned while function return type is `const T1&`, `b` may be converted to temporary `T1`.
- Returning reference to that temporary causes warning/dangling-reference risk.

Similar example:

```cpp
const int& fun(double& b) {
  return b; // temporary int is created, then returned by reference
}
```

Possible solutions:

1. Return by value.
2. Avoid returning such converted references.

```cpp
double a = 0.5;
```

## Common Template Error: Cannot Deduce Type

```cpp
template <typename T>
T* create() {
  return new T;
}

int main() {
  create<int>();    // OK
}
```

## Class Template

```cpp
template <typename T>
class List_Node {
public:
  explicit List_Node(const T& x) : data(x) {}

  T data;
  List_Node* next{nullptr};
};

// initialization
// List_Node<int> node(10);
```

Template with non-type parameter:

```cpp
template <typename T, int N>
class Container {
private:
  T data[N];
};
```

Remark: non-type parameter must be a compile-time constant expression.

## Compound Data Type and Reference to Array

```cpp
int arr[9] = {1, 2, 3, 4, 5, 6, 7, 8, 9};
int (&ref)[9] = arr; // reference to array

int arr2[3] = {1, 2, 3};

void fun(int (&r)[9]) {}
fun(arr); // OK: type is int[9]
```

<span style="color: red;">反例: fun(arr2); （int[3] cannot bind to int (&)[9]）</span>

Template version:

```cpp
template <typename T, int N>
int f(T (&x)[N]) {
  return N;
}
f(arr);  // T=int, N=9
f(arr2); // T=int, N=3
```

## Function Template vs Class Template

For function templates, compiler may deduce template arguments:

```cpp
int i = larger(4, 5);      // deduced as larger<int>
int j = larger<int>(7, 2); // explicit
```

For class templates, actual template arguments must be provided when creating objects.

```cpp
List_Node<int> node(2);
```

<span style="color: red;">反例: List_Node node; （template argument missing）</span>

## Separate Compilation and Templates

For non-template programs:

- Declaration in header (`.h`)
- Definition in source (`.cpp`)

For templates, definition must be visible at point of instantiation, so usually put declaration + definition in header.

Wrong pattern:

```cpp
// utils.h
template <typename T>
T add(T a, T b);

// utils.cpp
template <typename T>
T add(T a, T b) {
  return a + b;
}

// main.cpp
int main() {
  int x = add(3, 5);        // link/instantiation issue
  double y = add(3.14, 2.5);
}
```

Correct pattern:

```cpp
// utils.h
#pragma once

template <typename T>
T add(T a, T b) {
  return a + b;
}

// main.cpp
int main() {
  int x = add(3, 5);
  double y = add(3.14, 2.5);
}
```

## Remark: Assignment Expression Value

In C/C++, the value of assignment expression is the assigned value itself.

```cpp
// (a = 0) has value 0
if (0) {
  // always false
}

int a = 1;
if (a = 0) {
  // also false (but this is usually a bug)
}
```

## PBV in Polymorphism and Slicing

Passing derived object by value as base type may cause slicing.

```cpp
class B {};
class D : public B {};

void pbv(B b) {} // pass by value

int main() {
  D obj;
  pbv(obj); // D part is sliced away
}
```
