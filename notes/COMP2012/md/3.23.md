## Function Template

```cpp
template <typename T>
const T& larger(const T& a, const T& b) {
  return (a > b) ? a : b;
}

// Calls are all fine:
// larger(10, 20)
// larger(1.5, 2.5)
// larger(std::string("comp2011"), std::string("cpp"))
```

## Template Instantiation

Template definition is converted by compiler into actual code by substituting `T`.

- `T` is a formal template parameter.
- Different argument types produce separate instantiated functions.
- More types usually means larger executable size.

```cpp
void g(int a, int b) {}
g(10, 0.5); // OK: implicit conversion double -> int
```

<span style="color: red;">反例: larger(10, 0.5); （template deduction requires same T）</span>

For simple function template with one `T`, implicit type coercion between parameters is not used during deduction.

### If type conversion is needed

1. Manual conversion:

```cpp
larger(10.0, static_cast<double>(0.5)); // OK
```

2. Explicit template argument:

```cpp
larger<int>(10, 0.5);     // 0.5 -> 0
larger<double>(10, 0.5);  // 10 -> 10.0
```

3. Use multiple type parameters:

```cpp
template <typename T1, typename T2>
auto larger2(T1 a, T2 b) {
  return (a > b) ? a : b;
}

// larger2(10, 0.5);
// larger2(0.5, 10);
```

Note: many type combinations can generate many code fragments.

## Remarks on String Literals and Pointers

<span style="color: red;">反例: larger("comp2012", "c++") （may fail when template expects same array type）</span><br>
<span style="color: red;">原因: literal types are const char[9] and const char[4]</span>

```cpp
// One fix:
larger<std::string>("comp2012", "c++");
```

Comparing `const char*` directly compares addresses, not lexicographic content.

```cpp
const char* s3 = "microsoft";
const char* s4 = "apple";
larger(s3, s4); // compares pointer addresses by default
```

Address inspection:

```cpp
using namespace std;

cout << s3 << endl;                               // microsoft
cout << reinterpret_cast<const void*>(s3) << endl; // pointer address
```

## Template Specialization

```cpp
template <typename T>
const T& larger(const T& a, const T& b) {
  return (a > b) ? a : b;
}

template <>
const char* const& larger(const char* const& a, const char* const& b) {
  return (std::strcmp(a, b) > 0) ? a : b; // compare content
}
```

When `larger(s3, s4)` is called with `const char*`, the specialization is preferred because it matches better.

If `template <>` is removed, that version becomes an ordinary overload, and overload resolution prefers ordinary function over template when both match.

Why `const char* const&`?

- `const char*`: pointer to const char.
- `const char* const`: const pointer to const char.
- `const char* const&`: reference to a const pointer to const char.
